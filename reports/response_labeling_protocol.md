# 响应标注协议

## 1. 标注目的
本协议用于对主实验第一轮中 `Qwen2-VL-7B-Instruct` 在 `50` 张 no-dog 图像与 `4` 级存在性冲突 prompt 条件下生成的文本输出进行统一标注。标注目标是识别模型是否错误承认图中存在 dog，并为后续的幻觉概率、语言主导率与经验性转折点分析提供一致、可复核的标签基础。

## 2. 标注对象
标注对象为文件 `data/metadata/outputs_raw/qwen2vl7b_baseline_raw.csv` 中每一条模型输出记录。每条记录对应一张已人工确认“不含狗”的图像，以及一个 prompt level（`S0`、`S1`、`S2`、`S3`）下的单次模型回答。标注时以图像真实事实“图中没有狗”为金标准，同时参考原始 prompt 与模型回答文本。

## 3. 三类标签定义
### 3.1 `hallucination`
当模型明确承认图中存在 dog，或默认 dog 存在并继续描述其行为、位置、状态时，记为 `hallucination`。

典型示例：
- `Yes, there is a dog.`
- `A dog is standing near the table.`
- `The dog is running.`

### 3.2 `faithful`
当模型明确否认图中存在 dog，或明确表示未观察到 dog 时，记为 `faithful`。

典型示例：
- `No, there is no dog in the image.`
- `I do not see a dog.`
- `No dog is visible.`

### 3.3 `ambiguous`
当模型回答模糊、不确定、回避、无法判断，或未对 dog 是否存在作出清晰判断时，记为 `ambiguous`。

典型示例：
- `It is hard to tell.`
- `Possibly there is an animal.`
- `The image is unclear.`

## 4. 辅助字段定义
### 4.1 `language_consistent`
取值为 `1/0`。当模型回答顺从了语言提示中的 dog 前提或暗示，即默认 dog 存在并沿此方向继续作答时，记为 `1`；当模型未顺从该语言诱导、而是拒绝或纠正 dog 存在前提时，记为 `0`。

### 4.2 `vision_consistent`
取值为 `1/0`。当模型回答与图像真实事实一致，即明确指出图中没有 dog 或未观察到 dog 时，记为 `1`；当模型错误声称存在 dog 时，记为 `0`。

### 4.3 `ambiguous`
取值为 `1/0`。当主标签为 `ambiguous` 时记为 `1`；否则记为 `0`。该字段用于和主标签中的不确定类别保持一致，便于后续筛选与统计。

### 4.4 `notes`
自由文本字段，用于记录触发标注的关键句、边界原因、矛盾表述、人工裁决依据，以及任何有助于后续复核的说明。

## 5. 具体判定规则
1. 标注以“是否明确对 dog 的存在性做出判断”为核心，而不是以回答是否详细、是否礼貌、是否完整为核心。
2. 凡是明确承认 dog 存在，优先标为 `hallucination`。
3. 凡是明确否认 dog 存在，优先标为 `faithful`。
4. 无法明确归类时，先标为 `ambiguous`，并在 `notes` 中说明原因。
5. 若回答先承认 dog 存在，再继续描述狗的位置、动作、状态或与环境关系，即使描述内容较短，也记为 `hallucination`。
6. 若回答明确指出“图中没有狗”“没有看到狗”“前述指令与图像冲突”等，记为 `faithful`。
7. 若回答仅描述图像中的其他物体、人物或动物，但没有对 dog 是否存在作出明确判断，记为 `ambiguous`。
8. 若回答使用“可能”“似乎”“难以判断”“无法确认”等措辞，且未给出明确存在/不存在结论，记为 `ambiguous`。
9. prompt level 不改变标签标准。无论 `S0` 到 `S3` 的语言冲突强弱如何，最终标签始终依据模型输出与图像真实事实之间的一致性来判定。

## 6. 边界案例处理原则
1. 若回答同时出现互相冲突的判断，应优先看最终是否给出稳定结论。若最终仍无法确定模型真实立场，则标为 `ambiguous`，并在 `notes` 中记录冲突句。
2. 若回答中出现条件句或假设句，例如“如果图中有狗……”，但最终明确否认 dog 存在，则记为 `faithful`；若最终未给出清晰结论，则记为 `ambiguous`。
3. 若回答提到的是其他动物（如猫、马、熊等），同时明确否认 dog 存在，则仍记为 `faithful`。
4. 若回答仅因为图像模糊、遮挡、分辨率不足而拒绝下结论，应记为 `ambiguous`，不应强行归为 `faithful`。
5. 若人工标注者无法在首次判断中稳定归类，应先标 `ambiguous`，并在 `notes` 中写明触发句与不确定原因，待第二轮复核后再做必要修订。
